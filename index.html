Edukasi WSL Hub & Docker

Panduan langkah-demi-langkah untuk pemula Windows — setup WSL2 dan Docker dari kosong dengan bantuan AI-agent.

🖥️ CLI (opencode) 🌐 Web AI (ChatGPT/Gemini/Claude) 📴 Offline (Ollama+Gemma)

Apa Projek Ni?

Kebanyakan guide setup WSL2/Docker di luar sana anggap pembaca dah reti teknikal. Projek ni dibina khas untuk pemula total — orang yang baru pertama kali buka PowerShell — dan direka supaya boleh diikuti sambil dibantu AI-agent.

📚 Semua Section

Mula dari Section 01 kalau langsung tak pernah setup, atau terus ke mana-mana topik yang nak dipelajari.

01

Prasyarat & Enable WSL

Check versi Windows, enable Virtual Machine Platform, enable WSL, restart, dan BIOS virtualization.

02

Istilah Asas (Glossary)

WSL, distro, kernel, container, image, daemon, port, CLI — semua dijelaskan dengan analogi mudah.

03

WSL2 — Setup

Install WSL, setup Ubuntu, update package, dan verifikasi semuanya berjalan lancar.

04

WSL2 — Pengurusan & Sesi

Keluar terminal, shutdown WSL untuk bebaskan RAM, dan urus sesi Ubuntu.

05

Docker — Pemasangan

Pasang Docker Engine dalam Ubuntu WSL2 tanpa Docker Desktop — lebih ringan dan pantas.

06

Docker — Urus Container

Hidupkan Docker daemon, lari container Nginx, dan faham port forwarding.

07

Docker — Pembersihan

Hentikan, padam container, dan prune sistem untuk jimat ruang disk.

08

Masalah Lazim (Troubleshooting)

Ralat biasa: daemon tak jalan, permission denied, port conflict — dan cara selesaikannya.

09

Q&A

Soalan-soalan lazim: kenapa guna Docker Engine instead of Docker Desktop, apa jadi container masa laptop dimatikan, dan lain-lain.

10

Rujukan Pantas

Cheat sheet semua command penting — WSL, Docker, cleanup — dalam satu halaman.

💡
Tip: Section 01 dan 02 sedang dalam pembinaan. Sementara tu, boleh terus ke Section 03 kalau dah biasa dengan konsep asas.

🚀 Cara Guna

  1. Buka mana-mana section dari senarai di atas
  2. Ikut turutan 01 → 10 kalau baru pertama kali, atau terus ke topik yang nak dipelajari
  3. Setiap section ada bahagian ⚠️ Troubleshooting — kalau stuck, cari dulu di situ
  4. Guna AI-agent (ChatGPT, Claude, Gemini, atau opencode CLI) untuk bantuan tambahan

🧭 Falsafah Projek

  • Pemula dulu, teknikal kemudian — setiap istilah teknikal dijelaskan bila kali pertama disebut
  • Faham, bukan hafal — setiap command ada penjelasan "kenapa" bukan sekadar "macam mana"
  • Mobile-first — dioptimumkan untuk baca dari phone/Telegram
  • AI-agent-friendly — struktur projek direka untuk AI-agent boleh sambung kerja tanpa context berulang
