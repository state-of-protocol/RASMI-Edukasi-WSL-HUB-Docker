01 Prasyarat & Enable WSL

Sebelum boleh guna WSL2 atau Docker, pastikan komputer anda sudah memenuhi syarat minimum. Section ni akan bimbing anda langkah demi langkah.

📋 Kandungan Section Ini

  1. Semak Versi & Build Windows
  2. Enable Virtual Machine Platform & WSL
  3. Install WSL dengan Satu Command
  4. Restart Komputer
  5. Semak & Enable Virtualization dalam BIOS

1. Semak Versi & Build Windows

WSL2 memerlukan Windows 10 versi 2004 (build 19041) atau lebih baru, atau mana-mana versi Windows 11. Jom semak dulu versi Windows komputer anda.

Guna Command winver (Paling Mudah)

Tekan Win + R pada keyboard, taip winver, dan tekan Enter. Satu kotak dialog akan muncul — cari baris yang tulis "Version" atau "Versi".

Apa yang perlu dilihat: Pastikan version adalah 2004 atau lebih baru (contoh: 21H2, 22H2) atau mana-mana version Windows 11. OS Build pula perlu 19041 atau lebih tinggi.

🖼️ Screenshot: Kotak dialog winver — tunjuk Windows 10 Version 22H2, Build 19045

Guna Settings (GUI)

  1. Buka Settings (tekan Win + I)
  2. Pergi ke System → About
  3. Di bawah "Windows specifications", cari Edition, Version, dan OS Build

Guna PowerShell (Alternatif)

Buka PowerShell dan taip command ni untuk tengah versi dengan lebih detail.

(Get-CimInstance Win32_OperatingSystem).Caption

Command ni akan tunjuk nama penuh Windows (contoh: "Microsoft Windows 10 Pro").

(Get-CimInstance Win32_OperatingSystem).BuildNumber

Ini tunjuk nombor build. Jika 19041 atau lebih, anda boleh guna command wsl --install terus.

✅
Checkpoint: Kalau anda nampak version 2004+ dan build 19041+, komputer anda sudah boleh jalan WSL2. Terus ke langkah 2.
⚠️
Amaran: Kalau version Windows anda lebih lama dari 2004 (contoh: 1909 atau 1809), anda perlu update Windows dulu melalui Windows Update. WSL2 tidak akan berfungsi pada build yang terlalu lama.

2. Enable Virtual Machine Platform & Windows Subsystem for Linux

WSL2 perlukan dua komponen Windows yang perlu diaktifkan: Virtual Machine Platform dan Windows Subsystem for Linux. Ada dua cara — guna GUI atau PowerShell.

Cara A — Guna Windows Features (GUI)

Ini cara paling mudah untuk yang baru belajar.

  1. Buka Control Panel → Programs → Turn Windows features on or off
  2. ATAU tekan Win + R, taip optionalfeatures, tekan Enter
  3. Dalam senarai, tanda dua item ini:
    • Virtual Machine Platform
    • Windows Subsystem for Linux
  4. Klik OK

🖼️ Screenshot: Tetingkap Windows Features dengan "Virtual Machine Platform" dan "Windows Subsystem for Linux" ditanda

Cara B — Guna PowerShell (Alternatif)

Buka PowerShell sebagai Administrator — klik kanan icon PowerShell, pilih "Run as Administrator". Kemudian salin dan jalankan command berikut SATU PERSATU:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

Command ni enable Windows Subsystem for Linux tanpa restart dulu.

dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Command ni enable Virtual Machine Platform.

💡
Tip: Kalau anda guna Windows yang lebih baru (build 19041+), anda boleh guna command wsl --install di langkah 3 yang akan enable kedua-dua feature ni secara automatik. Tak perlu guna cara A atau B jika guna command tu.
✅
Checkpoint: Lepas langkah ni, anda akan nampak mesej "The operation completed successfully" untuk setiap command. Jangan tutup PowerShell — kita akan guna dia dalam langkah seterusnya.

3. Install WSL dengan Satu Command

Sekarang, kita install WSL dan Linux distro (Ubuntu) serentak.

Jalankan Command WSL Install

Dalam PowerShell (yang masih dibuka sebagai Administrator), jalankan command ni:

wsl --install

Command ni akan:

  • Enable dua feature Windows yang diperlukan (kalau belum)
  • Muat turun dan install Linux kernel
  • Set WSL2 sebagai versi lalai
  • Install Ubuntu distro

⚠️
Amaran: Command wsl --install hanya tersedia pada Windows build 19041+ (Windows 10 version 2004+ dan semua Windows 11). Kalau anda guna build lama, guna cara manual di Langkah 2 dan kemudian install distro dari Microsoft Store atau guna wsl --install -d Ubuntu.

Semak Hasil Installasi

Selepas selesai, anda boleh verify dengan command ni:

wsl --status

Command ni akan tunjuk status WSL — termasuk versi default dan kernel version.

wsl -l -v

Command ni tunjuk senarai distro yang telah diinstall dan versi WSL masing-masing.

✅
Checkpoint: Kalau wsl --status tunjuk "Default Version: 2" dan ada distro Ubuntu dalam senarai, setup WSL dah berjaya!

4. Restart Komputer

Lepas enable features dan install WSL, komputer anda perlu restart untuk proses selesai sepenuhnya.

⚠️
Amaran: Jangan skip restart. Tanpa restart, feature Virtual Machine Platform mungkin tidak aktif dan WSL2 tidak akan berfungsi dengan betul. Simpan semua kerja dulu sebelum restart.

Lepas restart, anda boleh terus ke Section 03 — WSL2 Setup untuk mula guna Ubuntu.

5. Semak & Enable Virtualization dalam BIOS

WSL2 perlukan ciri virtualisasi hardware yang disokong oleh processor komputer anda. Walaupun kebanyakan komputer moden sudah enable virtualization secara lalai, ada juga yang masih disabled.

Semak Status Virtualization dalam Windows

Sebelum masuk BIOS, semak dulu dari Windows:

  1. Tekan Ctrl + Shift + Esc untuk buka Task Manager
  2. Pergi ke tab Performance
  3. Klik pada CPU
  4. Cari baris Virtualization — kalau tulis "Enabled", anda tak perlu buat apa-apa lagi

🖼️ Screenshot: Task Manager → Performance → CPU — Virtualization: Enabled

✅
Checkpoint: Kalau Task Manager tunjuk "Virtualization: Enabled", terus ke Langkah 4 — Restart atau ke section seterusnya. BIOS tak perlu diusik.

Enable Virtualization dalam BIOS (Jika Perlu)

Kalau Task Manager tunjuk "Virtualization: Disabled", anda perlu enable kan dalam BIOS/UEFI. PERHATIAN: Cara masuk BIOS dan lokasi setting ni BERBEZA ikut jenama motherboard atau laptop anda.

Pertama, tahu versi Windows anda: Guna winver (Langkah 1 di atas) untuk tahu sama ada Windows 10 atau 11 — lepas itu ikut path yang betul di bawah.

Cara masuk BIOS (UEFI) — Windows 10:

  1. Buka Settings → Update & Security → Recovery
  2. Di bawah "Advanced startup", klik Restart now
  3. Lepas restart: Troubleshoot → Advanced options → UEFI Firmware Settings → Restart

Cara masuk BIOS (UEFI) — Windows 11:

  1. Buka Settings → System → Recovery
  2. Di sebelah "Advanced startup", klik Restart now
  3. Lepas restart: Troubleshoot → Advanced options → UEFI Firmware Settings → Restart
💡
Tip: Tak jumpa path di atas? Guna search dalam Settings — taip "Recovery" atau "Advanced startup" untuk cari terus.

Cara alternatif (semua versi Windows): Tekan dan tahan Shift sambil klik Restart pada Start Menu. Lepas tu ikut: Troubleshoot → Advanced options → UEFI Firmware Settings → Restart.

ATAU semasa komputer mula hidup (selepas restart biasa), tekan kekunci tertentu — biasanya Del (desktop), F2 (laptop), atau F10 (HP) — tapi ini berbeza ikut jenama.

Cari setting virtualization — nama yang biasa digunakan:

  • Intel: "Intel Virtualization Technology", "VT-x", "Vanderpool", atau "Virtualization"
  • AMD: "SVM Mode", "Secure Virtual Machine", atau "AMD-V"

Lokasi bias (tapi TIDAK universal):

  • Advanced → CPU Configuration
  • Overclocking → CPU Features
  • Security → Virtualization
  • Configuration → Virtualization Technology
💡
Tip: Menu BIOS berbeza ikut jenama. Guna panduan ni: Panduan rasmi Microsoft atau cari "enable virtualization BIOS [jenama laptop/motherboard anda]" di Google — contoh: "enable virtualization BIOS ASUS ROG", "enable virtualization BIOS Dell Inspiron".
⚠️
Amaran: Berhati-hati dalam BIOS — jangan ubah setting lain yang anda tidak faham. Cukup cari dan enable virtualization sahaja, kemudian tekan F10 (atau ikut arahan di skrin) untuk Save & Exit.

🔧 Troubleshooting

Kalau langkah di atas tak berjalan seperti yang dijangka, cuba semak isu-isu berikut:

❌
wsl --install gagal atau "command not found"

Ini bermakna Windows build anda terlalu lama (bawah 19041). Guna Windows Update untuk update ke versi terkini, atau guna cara manual di Langkah 2 untuk enable features guna DISM.

❌
WSL installation incomplete atau error 0x8007007e

Biasanya sebab virtualization belum dienable dalam BIOS. Semak Langkah 5 di atas untuk cara enable virtualization.

❌
Mesej "Please enable Virtual Machine Platform"

Walaupun sudah enable, restart belum dilakukan. Restart komputer dan cuba lagi. Kalau masih bermasalah, rujuk Troubleshooting penuh.
