02 Istilah Asas (Glossary)

Dalam dunia WSL dan Docker, banyak istilah teknikal yang mungkin bunyi asing. Section ni terangkan setiap istilah dengan bahasa mudah dan analogi, supaya anda faham konsep — bukan sekadar hafal command.

WSL — Windows Subsystem for Linux

WSL (Windows Subsystem for Linux) adalah ciri Windows yang membolehkan anda jalan Linux dalam Windows tanpa perlu dual-boot atau mesin maya yang berat.

💡
Analogi: Bayangkan Windows adalah sebuah rumah, dan Linux adalah sebuah bilik khas dalam rumah tu. Dengan WSL, anda boleh masuk ke bilik Linux tu, guna semua alat Linux, dan bila selesai, tutup pintu — tanpa perlu rumah lain (mesin maya) atau pindah rumah (dual-boot). Bilik ni guna electricity rumah yang sama — sebab tu WSL sangat ringan pada RAM.

Ada dua versi WSL:

  • WSL 1: Versi lama — translate system calls Linux kepada Windows. Cepat tapi tak semua app Linux boleh jalan.
  • WSL 2: Versi baru (disarankan) — guna kernel Linux sebenar dalam mesin maya ringan. Lebih serasi dengan Docker dan app Linux lain.

Distro — Distribution (Edaran Linux)

Distro (singkatan untuk distribution) adalah "perisa" Linux yang berbeza. Sama macam ada pelbagai jenis minuman kopi — americano, latte, cappuccino — semuanya kopi, tapi cara penyediaan dan rasanya lain. Linux pun macam tu: semua guna kernel yang sama, tapi cara dia dibungkus dan alat-alat yang disertakan berbeza.

Contoh distro popular:

  • Ubuntu — paling mesra pemula, banyak tutorial online. Ini yang akan kita guna dalam panduan ni.
  • Debian — stabil, ringan. Sesuai untuk server.
  • Fedora — sentiasa ada teknologi terkini.
  • Alpine — sangat kecil (saiz MB, bukan GB). Popular untuk container Docker.

Dalam WSL, anda boleh install beberapa distro serentak dan bertukar antara mereka. Tapi untuk pemula, cukup guna Ubuntu dulu.

Kernel — Bahagian Teras Sistem Operasi

Kernel adalah "otak" sistem operasi. Dia yang uruskan komunikasi antara hardware (CPU, RAM, disk) dan software (app yang anda guna).

💡
Analogi: Kernel macam pelayan restoran. Bila anda (app) nak air (data), anda cakap dengan pelayan. Pelayan pergilah ke dapur (hardware), ambil air, dan bawa balik kepada anda. Anda tak perlu tahu apa jadi di dapur — pelayan yang uruskan semua.

Dalam WSL 2, Microsoft guna kernel Linux sebenar yang telah diubah suai sedikit untuk jalan dalam Windows. Kernel ni diupdate secara automatik melalui Windows Update.

Container — Kotak Ringan untuk App

Container adalah "kotak" yang mengandungi app anda + semua yang app tu perlukan untuk jalan (library, configuration, dependencies).

💡
Analogi: Container macam bekas Tupperware yang berisi makanan lengkap. Bekas tu ada makanan, sudu, serbet — semuanya sekali. Bila nak makan, buka bekas, siap. Tak perlu cari pinggan, tak perlu susun. Bila habis, tutup balik dan simpan. Tak ada kesan — sebab semuanya dalam bekas tu saja. Container dalam Docker pun sama: app + semua keperluannya dalam satu bekas.

Beza dengan mesin maya (VM): Container jauh lebih ringan sebab dia kongsi kernel dengan tuan rumah, bukan jalan OS penuh sendiri. Sebab tu container boleh mula dalam saat, bukan minit.

Image — Blueprint untuk Container

Image adalah "cetak biru" (blueprint) untuk container. Kalau container adalah rumah yang sudah siap, image adalah pelan rumah — belum siap, tapi mengandungi semua arahan untuk bina rumah tu.

💡
Analogi: Image macam resepi kek. Resepi tu tak boleh dimakan (sebab tulisan di atas kertas), tapi guna resepi tu anda boleh buat banyak kek (container) yang sama. Dalam Docker, image adalah resepi, dan container adalah kek yang dah siap.

Anda boleh download image orang lain dari Docker Hub (macam kedai resepi online), atau bina image sendiri guna Dockerfile.

✅
Checkpoint: Ingat: docker pull = muat turun resepi (image). docker run = guna resepi untuk buat kek (container).

Daemon — Servis Latar Belakang

Daemon (sebut: "de-mon") adalah program yang jalan di latar belakang, tanpa perlu anda buka tetingkap atau berinteraksi langsung.

💡
Analogi: Daemon macam penjaga malam di bangunan. Dia duduk di kaunter depan, 24 jam, tunggu orang datang. Bila ada tetamu (app), dia sambut dan uruskan. Bila takde tetamu, dia diam — tapi tetap bersedia. Docker daemon pun sama: dia tunggu arahan docker (contoh: docker run), dan dia akan laksanakan.

Dalam setup kita, daemon Docker dipanggil dockerd. Setiap kali lepas restart laptop, anda kena hidupkan daemon ni dengan sudo service docker start sebelum guna Docker.

Port — Pintu untuk Data

Port adalah "pintu" maya pada komputer yang membenarkan data masuk dan keluar.

💡
Analogi: Bayangkan komputer anda adalah sebuah apartmen bertingkat. Alamat IP adalah alamat apartmen tu (contoh: 123, Jalan SS2). Tapi dalam apartmen tu ada banyak unit — unit 80 untuk web, unit 22 untuk SSH, unit 5432 untuk database. Nombor port adalah nombor unit apartmen. Bila anda tak suruh container guna port 8080, anda sebenarnya kata: "Buka unit 8080 untuk container ni".

Port biasa yang penting:

  • 80 — Web (HTTP) — lalai untuk laman web
  • 443 — Web secure (HTTPS)
  • 22 — SSH (remote access)
  • 5432 — PostgreSQL database
  • 8080 — Port alternatif untuk web (selalu guna untuk testing)

Dalam Docker, -p 8080:80 maksudnya "Buka pintu 8080 di komputer saya, sambung ke pintu 80 dalam container".

CLI / Terminal — Cakap dengan Komputer Guna Teks

CLI (Command Line Interface) atau terminal adalah cara berinteraksi dengan komputer dengan menaip arahan teks, bukan klik ikon atau butang.

💡
Analogi: Kalau GUI (graphical interface) macam memandu kereta automatik — senang, semua ada butang, tapi terhad. CLI pula macam memandu kereta manual — nampak susah pada mulanya, tapi lepas biasa, anda boleh buat lebih banyak benda dengan lebih pantas. Anda cakap terus pada enjin, tanpa tunggu GUI.

Dua terminal yang anda akan guna:

  • PowerShell — terminal Windows. Guna untuk command berkaitan Windows dan WSL.
  • WSL Ubuntu — terminal Linux dalam WSL. Guna untuk command Linux dan Docker.

Jangan risau kalau nampak banyak command — panduan ni akan bagi setiap command dengan penjelasan apa dia buat. Anda tak perlu hafal, cukup faham konsep.

✅
Checkpoint: Ingat: PowerShell = command untuk urus Windows/WSL. WSL Ubuntu = command untuk urus Docker dan app Linux. Jangan campur aduk — nanti command tak jalan!
