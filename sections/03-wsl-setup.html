1. Setup Akaun Ubuntu (First-Time Setup)

Lancarkan Ubuntu

Ada dua cara untuk buka Ubuntu dalam WSL buat kali pertama:

  • Cari "Ubuntu" dalam Start Menu dan klik icon dia, ATAU
  • Buka PowerShell dan taip wsl

Pada kali pertama sahaja, Ubuntu akan mengambil beberapa minit untuk memasang dan mengkonfigurasi diri. Anda akan nampak mesej seperti "Installing, this may take a few minutes..." — ini normal, jangan tutup tetingkap.

Cipta Nama Pengguna & Kata Laluan

Selepas proses selesai, Ubuntu akan meminta anda mencipta akaun UNIX (berasingan daripada akaun Windows anda). Anda akan nampak dua prompt:

  1. Nama pengguna (username): Biasanya dipaparkan sebagai Enter new UNIX username: atau Enter new username:. Taip nama pilihan — guna huruf kecil sahaja, tanpa ruang. Contoh: rafee.
  2. Kata laluan (password): Seterusnya New password: — taip kata laluan. Ia tidak akan kelihatan semasa ditaip (tiada asterisk atau titik) — ini normal. Ulang sekali lagi di Retype new password: untuk pengesahan.
⚠️
Penting: Nama pengguna dan kata laluan ini adalah untuk akaun Linux dalam Ubuntu, BUKAN akaun Windows anda. Ingat username ini — anda akan gunakannya setiap kali buka terminal WSL. Kalau lupa password, anda kena reset dari PowerShell guna wsl --user root.
💡
Tip: Nama pengguna UNIX boleh sama dengan username Windows anda (contoh: rafee) atau berbeza — terpulang. Tapi password UNIX dan password Windows adalah berasingan — menukar salah satu tidak akan menukar yang lain.

Checkpoint — Selesai Setup

Selepas selesai, anda akan lihat prompt terminal yang menunjukkan username + hostname, contohnya:

WSL Ubuntu 
rafee@DESKTOP-ABC123:~$
✅
Checkpoint: Kalau nampak prompt macam namaanda@namakomputer:~$, maknanya Ubuntu sudah sedia dan anda kini berada dalam terminal Linux. Hostname (selepas @) mungkin berbeza — ia adalah nama komputer Windows anda.

2. Kemas Kini Pakej Ubuntu

Kemas Kini Senarai Pakej

Sebelum guna Ubuntu untuk apa-apa, biasakan diri untuk kemas kini senarai pakej dahulu. Command ni akan muat turun maklumat versi terkini semua pakej dari repositori Ubuntu.

WSL Ubuntu 
sudo apt update

Masukkan kata laluan UNIX anda apabila diminta. Output akan menunjukkan senarai repositori yang sedang dikemas kini. Jika selesai tanpa ralat, anda sudah bersedia untuk langkah seterusnya.

💡
Tip: Kali pertama guna sudo, anda akan nampak mesej seperti "is not in the sudoers file" — TAPI ini hanya jika anda buka Ubuntu sebagai root. Jika anda setup akaun biasa (seperti langkah 1), sudo akan berfungsi terus dan hanya minta password.

Tingkatkan Semua Pakej

Selepas senarai pakej dikemas kini, jalankan command ni untuk naik taraf semua pakej yang sudah lapuk kepada versi terkini.

WSL Ubuntu 
sudo apt upgrade -y

Proses ini akan memuat turun dan memasang pakej yang lebih baru. Masa yang diperlukan bergantung pada jumlah pakej yang perlu dikemas kini dan kelajuan internet anda. Flag -y bermaksud "yes" — ia akan menjawab "Ya" secara automatik kepada sebarang pengesahan.

✅
Checkpoint: Lepas selesai, anda akan kembali ke prompt biasa tanpa sebarang ralat. Ini petanda Ubuntu sudah dikemas kini dan bersedia untuk kegunaan seterusnya.

3. Verifikasi Pemasangan WSL

Senarai Distro dengan Versi WSL

Untuk pastikan Ubuntu berjalan dengan WSL 2, jalankan command ni dalam PowerShell (bukan dalam WSL):

PowerShell 
wsl -l -v

Command ni akan paparkan senarai distro yang dipasang, status, dan versi WSL dalam bentuk jadual dengan tiga lajur: NAME (nama distro), STATE (Running / Stopped), dan VERSION (1 atau 2).

Gantikan -l dengan --list dan -v dengan --verbose jika suka guna bentuk panjang: wsl --list --verbose.

✅
Checkpoint: Cari nama distro Ubuntu anda dalam output. Pastikan lajur VERSION menunjukkan angka 2. Jika tunjuk 1, rujuk wsl --set-version Ubuntu 2 untuk tukar. Jika tiada langsung distro dalam senarai, pemasangan mungkin tidak lengkap — cuba wsl --install semula.

Status Keseluruhan WSL

Guna command ni untuk lihat gambaran penuh status WSL — termasuk versi lalai, tarikh kemaskini kernel, dan maklumat sistem:

PowerShell 
wsl --status

Output akan menunjukkan maklumat seperti versi lalai WSL (sepatutnya 2) dan status kernel Linux. Tiada "lulus/gagal" yang jelas — cuma pastikan tiada mesej ralat dalam output.

⚠️
Nota: Output sebenar bagi wsl -l -v dan wsl --status mungkin kelihatan sedikit berbeza bergantung pada versi WSL dan Windows yang digunakan. Format asas (tiga lajur untuk -l -v) adalah konsisten, tetapi susun atur dan maklumat tambahan mungkin berubah. Yang penting: distro Ubuntu wujud, State = Running (atau Stopped — asalkan wujud), dan Version = 2.

🔧 Troubleshooting

❌
wsl: command not found

Ini bermakna WSL belum dipasang atau enable. Rujuk Section 01 — Prasyarat untuk langkah enable dan install WSL. Pastikan anda dah restart lepas enable features.

❌
WSL 2 requires an update — kernel update needed

Kernel WSL 2 perlu dikemas kini secara manual untuk kali pertama. Muat turun dan jalankan pakej kemas kini kernel WSL2 dari Microsoft. Selepas itu, set WSL 2 sebagai lalai: wsl --set-default-version 2.

❌
sudo: command not found atau dpkg: error processing package

Ini jarang berlaku, tapi jika apt update gagal dengan ralat repositori, Cuba jalankan sudo apt update --fix-missing atau restart WSL dengan wsl --shutdown dan buka semula. Rujuk Section 08 — Troubleshooting penuh untuk isu lain.
