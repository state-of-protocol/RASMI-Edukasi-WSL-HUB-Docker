04 WSL2 — Pengurusan & Sesi

Arahan asas untuk mengawal sesi terminal Ubuntu dan penutupan sistem bagi menjimatkan memori RAM.

📋 Kandungan Section Ini

  1. Keluar Terminal (Pintu Sahaja)
  2. Matikan Seluruh WSL & Bebaskan RAM

1. Keluar Terminal (Pintu Sahaja)

Command ni adalah cara paling asas untuk keluar dari sesi terminal WSL Ubuntu semasa. Ia hanya menutup terminal — proses WSL dan Ubuntu tetap berjalan di latar belakang.

WSL Ubuntu 
exit
💡
Beza exit vs tutup window: Bila taip exit, anda keluar secara sopan — shell ditutup dengan betul, dan sebarang proses latar belakang yang masih hidup diberitahu. Kalau hanya tutup window terminal (klik X), proses mungkin tidak ditutup dengan sempurna.
⚠️
Ingat: exit hanya tutup terminal. WSL Ubuntu masih berjalan dan menggunakan RAM. Kalau nak bebaskan RAM sepenuhnya, gunakan wsl --shutdown (langkah seterusnya).

2. Matikan Seluruh WSL & Bebaskan RAM

Command ni mematikan keseluruhan mesin WSL — semua distro (Ubuntu, dan lain-lain) akan dihentikan serta-merta. Ini akan membebaskan RAM yang digunakan oleh WSL.

PowerShell 
wsl --shutdown
⚠️
Amaran: wsl --shutdown akan menghentikan SEMUA distro WSL secara paksa. Simpan kerja anda dalam WSL sebelum jalankan command ni. Selepas shutdown, anda perlu buka semula Ubuntu (Start Menu atau wsl) dan jalankan sudo service docker start jika guna Docker.
✅
Bila nak guna? Gunakan wsl --shutdown bila:
  • Nak bebaskan RAM untuk aplikasi berat (contoh: gaming, video editing)
  • Nak reset WSL yang bermasalah (container tak boleh stop, network issue)
  • Lepas selesai guna WSL untuk hari itu

🔧 Troubleshooting

❌
wsl --shutdown: The term 'wsl' is not recognized

Ini bermakna anda menjalankan command dalam terminal WSL Ubuntu, bukan PowerShell. wsl hanya boleh dijalankan dalam PowerShell atau Command Prompt Windows. Buka PowerShell dan cuba semula.

❌
Lepas shutdown, container Docker hilang atau tak boleh start

Ini normal. Docker daemon dimatikan bersama WSL. Buka semula Ubuntu, kemudian jalankan sudo service docker start untuk hidupkan semula daemon Docker. Container anda masih ada — guna docker start nama-container untuk hidupkan semula.
