05 Docker — Pemasangan

Langkah rasmi memasang Docker Engine di Ubuntu WSL2 tanpa memerlukan aplikasi berat Windows Docker Desktop.

📋 Kandungan Section Ini

  1. Pemasangan Repositori & Enjin Docker
  2. Kebenaran Akses Bukan-Root

1. Pemasangan Repositori & Enjin Docker

Ikut langkah ni untuk setup repositori rasmi Docker dan pasang enjin Docker dalam persekitaran WSL Ubuntu. Semua command ni dijalankan dalam terminal WSL (Ubuntu).

WSL Ubuntu 
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg lsb-release
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
💡
Apa script ni buat?
  1. apt update && apt upgrade — pastikan pakej Ubuntu terkini
  2. ca-certificates curl gnupg lsb-release — alatan asas untuk muat turun GPG key
  3. mkdir -p /etc/apt/keyrings — direktori untuk keyring Docker
  4. curl ... | gpg --dearmor — muat turun dan simpan GPG key rasmi Docker
  5. echo ... | tee — tambah repositori Docker ke senarai apt
  6. apt update — refresh senarai pakej dengan repositori baru
  7. apt install docker-ce ... — pasang enjin Docker dan plugin

2. Kebenaran Akses Bukan-Root (Tanpa Sudo)

Selepas selesai pemasangan, beri kebenaran kepada user biasa (bukan root) untuk guna command Docker tanpa perlu taip sudo setiap kali.

WSL Ubuntu 
sudo usermod -aG docker $USER
⚠️
Penting: Selepas jalankan command ni, log out dan log in semula atau restart WSL terminal untuk group baru berkuat kuasa. Kalau tak, anda akan nampak ralat permission denied walaupun dah tambah user ke group docker.
💡
Verify: Lepas restart terminal, jalan docker ps tanpa sudo. Kalau output senarai container (mungkin kosong) tanpa ralat, maknanya setup berjaya!

🔧 Troubleshooting

❌
gpg: cannot open '/etc/apt/keyrings/docker.gpg': Permission denied

Pastikan anda guna sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg. Kalau masih gagal, cuba buat direktori dulu: sudo mkdir -p /etc/apt/keyrings.

❌
docker: permission denied walaupun dah guna sudo usermod

Kumpulan docker hanya akan berkuat kuasa untuk sesi shell baru. Tutup terminal WSL dan buka semula. Atau jalan newgrp docker untuk aktifkan dalam sesi semasa.
