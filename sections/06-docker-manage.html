06 Docker — Urus Container

Cara menghidupkan servis Docker latar belakang serta memulakan container pelayan web baharu.

📋 Kandungan Section Ini

  1. Hidupkan Servis Daemon Docker
  2. Luncurkan Server Ujian Nginx
  3. Semak Senarai Container Yang Aktif

1. Hidupkan Servis Daemon Docker

Selepas reboot atau kali pertama guna Docker, daemon Docker perlu dihidupkan secara manual. Servis ini berjalan di latar belakang dan menguruskan semua container.

WSL Ubuntu 
sudo service docker start
💡
Tip: Lepas jalan command ni, verify dengan sudo service docker status. Kalau output docker is running, daemon dah aktif dan sedia guna.
⚠️
Wajib setiap kali reboot: Docker daemon tak di-set untuk auto-start dalam WSL secara lalai. Jadi selepas setiap restart Windows atau WSL, ingat jalankan sudo service docker start dulu sebelum guna mana-mana command Docker.

2. Luncurkan Server Ujian Nginx

Guna command ni untuk melancarkan container Nginx — pelayan web ringan — dan pasangkannya ke port 8080 komputer anda. Container akan berjalan di latar belakang.

WSL Ubuntu 
docker run -d -p 8080:80 --name web-node nginx
💡
Apa flags ni buat?
  • -d — detached, jalan di latar belakang (terminal bebas)
  • -p 8080:80 — port 8080 komputer → port 80 dalam container
  • --name web-node — nama container, senang rujuk nanti
  • nginx — nama imej yang nak digunakan
✅
Checkpoint: Buka browser dan pergi ke http://localhost:8080. Kalau nampak halaman "Welcome to nginx", container dah berjalan dengan jayanya!

3. Semak Senarai Container Yang Aktif

Untuk lihat container mana yang sedang berjalan sekarang, guna command ni. Ia hanya menunjukkan container yang aktif — container yang dihentikan tak akan muncul.

WSL Ubuntu 
docker ps
💡
Nak nampak semua container (termasuk yang berhenti)? Guna docker ps -a. Flag -a bermaksud "all" — ia akan senaraikan semua container, termasuk yang dah stop.

🔧 Troubleshooting

❌
docker: Cannot connect to the Docker daemon

Daemon Docker belum hidup. Jalankan sudo service docker start dan cuba semula.

Rujuk Section 08 — Troubleshooting penuh untuk isu lain.

❌
docker: Error response from daemon — port 8080 already in use

Port 8080 dah digunakan oleh container atau aplikasi lain. Tukar port: guna -p 8081:80 dan cuba semula.

❌
docker: Error response from daemon — Conflict, container name "/web-node" already exists

Container dengan nama web-node dah wujud. Padam dulu: docker rm -f web-node, atau guna nama lain: --name web-node-baru.
