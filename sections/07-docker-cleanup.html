07 Docker — Pembersihan

Apabila projek selesai, bersihkan container dan sistem storan agar cakera NVMe tidak dipenuhi cache.

📋 Kandungan Section Ini

  1. Hentikan Server Berjalan
  2. Padam Container Secara Paksa
  3. Padam Bersih Semua Cache & Imej Tak Digunakan

1. Hentikan Server Berjalan

Sebelum padam container, pastikan ia dihentikan dengan selamat. Guna command ni untuk hentikan container tanpa memadamkannya — container masih boleh dihidupkan semula nanti.

docker stop nama-container

2. Padam Container Secara Paksa

Kalau nak padam container terus (tak perlu simpan), guna command ni. Flag -f bermaksud "force" — dia akan padam walaupun container masih berjalan.

docker rm -f nama-container

Gantikan nama-container dengan nama container yang hendak dipadam. Contoh: docker rm -f web-node.

⚠️
Amaran: docker rm -f akan padam container secara kekal — data di dalamnya akan hilang melainkan anda guna volume yang dipasang (mounted volume) dari luar container.

3. Padam Bersih Semua Cache, Imej & Volume Tak Digunakan

Lepas selesai semua projek, jalankan command ni untuk bersihkan keseluruhan sistem Docker — imej yang tak guna, container lama, volume terbiar, dan network cache. Ini sangat membantu untuk jimatkan ruang disk NVMe.

docker system prune -a --volumes -f
💡
Tip: Command ni gabungan flag:
  • -a — padam semua imej tak guna (termasuk yang gantung/dangling)
  • --volumes — padam juga volume yang tak dipakai oleh mana-mana container
  • -f — force, jangan tanya confirm (skip prompt)
⚠️
Amaran: Command ni tidak boleh undo. Pastikan anda dah backup apa-apa data penting sebelum jalan. Volume yang dipadam akan hilang selama-lamanya.

🔧 Troubleshooting

❌
rm: cannot remove — container masih running

Kalau docker rm tanpa -f gagal, maknanya container masih berjalan. Guna docker stop nama-container dulu, atau tambah flag -f untuk force remove.

❌
docker: permission denied

Kalau nampak mesej ini, bermakna user anda belum ditambah ke group docker. Rujuk Troubleshooting untuk penyelesaian.
