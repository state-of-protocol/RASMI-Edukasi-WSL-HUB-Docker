08 Masalah Lazim (Troubleshooting)

Panduan menyelesaikan ralat diagnostik yang kerap ditemui semasa mengendalikan Docker Engine di atas WSL2.

📋 Isu dalam Section Ini

  1. Docker daemon tidak berjalan
  2. Sekatan kebenaran akses (Permission Denied)
  3. Konflik Port (Port Already Allocated)

❌ Isu 1: Docker daemon tidak berjalan

Ralat yang muncul:

Cannot connect to the Docker daemon at unix:///var/run/docker.sock

Ralat ni bermakna Docker daemon (servis latar belakang) belum dihidupkan. Ini biasa berlaku selepas restart laptop — WSL tidak automatically start Docker daemon.

Penyelesaian (Jalankan dalam WSL Ubuntu):

sudo service docker start
💡
Tip: Lepas jalan command ni, semak dengan sudo service docker status untuk pastikan ia sudah berjalan. Output sepatutnya "docker is running".

❌ Isu 2: Sekatan kebenaran akses (Permission Denied)

Ralat yang muncul:

Anda terpaksa menaip sudo di hadapan setiap command docker. Contoh: sudo docker ps bukannya docker ps.

Punca: User anda belum ditambah ke group docker. Docker daemon hanya boleh diakses oleh user dalam group docker atau root.

Penyelesaian (Daftar group dan restart terminal):

sudo usermod -aG docker $USER
⚠️
Amaran: Lepas jalankan command ni, anda perlu log out dan log in semula (atau restart WSL terminal) untuk perubahan group berkuat kuasa. Kalau tak, anda masih akan nampak permission denied.
✅
Checkpoint: Lepas restart terminal, cuba docker ps tanpa sudo. Kalau tak keluar ralat, penyelesaian berjaya!

❌ Isu 3: Konflik Port (Port Already Allocated)

Ralat yang muncul:

Port hantaran (contoh: 8080) telah digunakan oleh tugasan atau kontena lain. Biasanya berlaku bila anda cuba docker run -p 8080:80 tapi port 8080 sudah dipakai.

Sebab: Setiap port pada komputer Windows hanya boleh digunakan oleh satu proses pada satu masa. Kalau container lama masih guna port 8080, container baru tak boleh guna port yang sama.

Penyelesaian (Tukar port hos bahagian hadapan ketika meluncurkan kontena):

docker run -d -p 8081:80 --name web-node-baru nginx
💡
Tip: Guna port lain macam 8081, 8082, atau 3000. Apa-apa port yang free. Atau hentikan container lama dulu: docker stop web-node dan docker rm web-node, baru guna port 8080 balik.
