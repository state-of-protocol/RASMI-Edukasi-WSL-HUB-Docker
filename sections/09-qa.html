09 Q&A

Fahami logik pengendalian sistem dan struktur hos bagi pemantapan teori.

📋 Soalan dalam Section Ini

  1. Kenapa guna Docker Engine dalam WSL, bukan Docker Desktop?
  2. Adakah container terpadam bila tutup laptop?
  3. Macam mana Windows boleh akses localhost:8080 container?

1. Mengapa pasang Docker dalam WSL berbanding guna Docker Desktop?

Memasang Docker Engine terus di dalam Ubuntu WSL jauh lebih ringan kerana ia tidak memuatkan aplikasi antara muka (GUI) Windows yang berat seperti Docker Desktop. Ini menjimatkan penggunaan RAM sistem laptop secara drastik serta memberikan kelajuan prestasi penulisan disk I/O yang hampir sama seperti pelayan Linux sebenar.

💡
Ringkasan: Docker Engine dalam WSL = ringan, jimat RAM, prestasi tinggi. Docker Desktop = senang guna (ada GUI) tapi berat — lebih sesuai untuk pengguna yang tak selesa dengan command line.

2. Adakah container terpadam automatik bila tutup terminal atau matikan laptop?

Tidak. Container hanya akan bertukar status kepada "Stopped" (ditutup). Ia tidak akan dipadam. Apabila anda membuka semula terminal dan menghidupkan servis Docker daemon, anda boleh menjalankan command berikut untuk menyambung semula container yang telah dihentikan:

docker start nama-container

Data dalam container juga kekal — ia tidak hilang semasa ditutup. Tapi data hanya akan kekal selagi container tidak dipadam. Kalau anda padam container dengan docker rm, data di dalamnya akan hilang (kecuali jika disimpan dalam volume berasingan).

✅
Checkpoint: Ingat: docker stop = tidurkan container (boleh hidup balik). docker rm = padam terus container (data hilang).

3. Macam mana Windows boleh akses localhost:8080 sedangkan container dalam Linux?

Ini dibantu oleh ciri auto port forwarding (pemetaan port automatik) yang dibina dalam WSL2. Sistem WSL2 secara automatik memetakan sebarang port rangkaian pelayan yang diaktifkan di dalam Linux terus ke antara muka rangkaian localhost Windows hos anda.

Maksudnya: bila container berjalan dalam WSL dan dengar port 80, WSL2 akan sambungkan port tu ke port yang sama pada Windows. Jadi bila anda buka http://localhost:8080 (atau port yang anda tetapkan dengan -p), Windows akan hantar trafik terus ke container dalam WSL — tanpa sebarang konfigurasi tambahan.

💡
Analogi: WSL2 macam jambatan automatik antara rumah Windows dan rumah Linux. Bila ada trafik di port tertentu, jambatan ni buka laluan tanpa anda perlu bangun dan buka pintu manual.

🔧 Troubleshooting

❌
Container "Stopped" tapi tak boleh start balik?

Pertama, pastikan Docker daemon sedang berjalan. Jalankan sudo service docker status untuk semak. Kalau tak jalan, sudo service docker start dulu. Lepas tu baru docker start nama-container.
